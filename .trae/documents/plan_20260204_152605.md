# 修复 Steam 最新动态显示为 1970 的问题

## 问题分析
用户反馈游戏卡片中的“最新动态”变成了 1970年。经排查，原因为：
1. **时间戳为 0**: 显示为 1970/1/1 是因为后端 API 返回的时间戳为 0，或者请求失败导致默认值为 0。
2. **代理失效**: 目前使用的首选代理 `corsproxy.io` 对 Steam API 返回了 403 Forbidden 错误，导致获取新闻数据的请求失败。
3. **并发过高**: 代码中设置的并发请求数为 12，导致备用代理 `codetabs` 可能触发了速率限制 (Rate Limit) 而失败。
4. **数据覆盖逻辑缺陷**: 当 API 请求失败返回 0 时，系统直接用 0 覆盖了原有的有效时间戳。

## 实施计划

### 1. 优化代理配置 (`src/services/steamApi.ts`)
- 将 `corsproxy.io` 降级或移除（因其目前不可用）。
- 将 `codetabs` 提升为首选代理。
- 增加对代理返回数据的校验。

### 2. 调整并发控制 (`src/store/useGameStore.ts`)
- 将批量刷新时的并发数 (`concurrency`) 从 **12** 降低到 **5**。
- 这样可以避免触发 `codetabs` 等公共代理服务的速率限制，提高请求成功率。

### 3. 修复数据更新逻辑 (`src/store/useGameStore.ts`)
- 在 `fetchGameData` 函数中增加校验：只有当新获取的 `lastUpdate` 大于 0 时，才更新该字段。
- 如果 API 请求失败返回 0，则保留原有的 `lastUpdate` 时间戳，防止显示回退到 1970 年。

### 4. 验证
- 修改后，重新运行刷新功能，观察“最新动态”是否恢复正常（需等待一次成功的网络请求）。
- 确认“在线人数”等其他数据依然正常更新。
