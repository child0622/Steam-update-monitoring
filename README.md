## ✨ 项目简介 (Introduction)

**Steam 游戏更新监控助手** 是一个基于 Web 的本地监控面板。它可以帮你自动追踪你关注的 Steam 游戏，一旦检测到版本更新、新闻发布，会立即通过系统弹窗通知你。

哪怕你正在浏览网页或处理工作，只要它在后台运行，你就不会错过任何游戏动态。

### 核心亮点
*   **零失败机制**：独创的“死磕模式”，网络超时会自动降速并无限重试，直到成功。
*   **无需后端**：纯前端实现，利用公共 CORS 代理直连 Steam API，开箱即用。
*   **完全本地化**：所有数据保存在你的浏览器中，支持 JSON 备份/恢复，隐私安全。

---

## 🚀 功能特性 (Features)

### 1. ⚡ 智能更新系统
*   **手动刷新**：一键检查所有游戏更新，支持高并发请求。
*   **自动巡航**：默认每 **15分钟** 自动静默检查一次，有更新才打扰，无更新零干扰。
*   **无限重试 (Dead-lock Mode)**：
    *   遇到网络波动？系统会自动降低并发数。
    *   依然失败？系统切换为串行（一个接一个）模式。
    *   绝不放弃，直到获取到数据为止（无效 AppID 除外）。

### 2. 🔔 沉浸式通知
*   **系统级弹窗**：调用 Windows/Mac 原生通知中心。
*   **点击跳转**：看到更新通知，点击卡片即可自动聚焦回监控页面。
*   **更新摘要**：多款游戏同时更新时，会自动汇总为一条摘要通知，避免弹窗轰炸。

### 3. 🛡️ 数据安全与备份
*   **自动保存**：所有添加的游戏 ID 实时保存到本地存储 (LocalStorage)。
*   **一键备份**：点击顶部的“备份数据”按钮，生成 JSON 文件。
*   **一键恢复**：更换电脑或清除缓存后，点击“恢复数据”即可完美还原。

### 4. 🌐 极致体验
*   **全中文界面**：贴心的中文提示与错误引导。
*   **详细进度**：实时显示刷新进度（如：`正在刷新: 5 / 20`）。
*   **快速入口**：卡片内置“商店页面”按钮，一键直达 Steam Store。

## 🛠️ 本地部署指南 (Installation)

只需三步，即可在你的电脑上运行。

### 环境准备
请确保你的电脑已安装 [Node.js](https://nodejs.org/) (建议 v16 或更高版本)。

### 第一步：下载代码
```bash
git clone https://github.com/child0622/Steam-update-monitoring.git
cd steam-monitor-v2
```

### 第二步：安装依赖
在项目根目录下打开终端，运行：
```bash
npm install
```

### 第三步：启动项目
```bash
npm run dev
```
看到 `Local: http://localhost:5173/` 提示后，按住 `Ctrl` 点击链接即可打开使用。

## 📖 使用手册 (User Guide)

### 1. 添加游戏 ➕
1.  点击页面左上角的输入框。
2.  输入游戏的 **AppID**（例如 `2357570` 是《守望先锋2》）。
    > *如何获取 AppID？打开 Steam 商店页面，网址链接中 `app/` 后面的数字就是 ID。*
3.  点击 **“添加”** 按钮。
    *   支持批量导入！输入多个 ID 用逗号分隔即可。

### 2. 检查更新 🔄
*   **手动模式**：点击右上角的 **“刷新列表”** 按钮。
    *   右下角会弹出进度条，显示当前进度。
    *   如果发现更新，会弹出系统通知，并显示“更新摘要”弹窗。
*   **自动模式**：保持网页开启（可以最小化），系统每 15 分钟自动运行一次。

### 3. 管理游戏 🎮
*   **查看详情**：卡片显示游戏封面、名称、上次更新时间、当前在线人数。
*   **访问商店**：点击卡片底部的 **“商店页面”** 图标。
*   **单独刷新**：点击卡片底部的 **“刷新”** 图标，只更新这一个游戏。
*   **删除游戏**：点击卡片右上角的 **“X”** 按钮。

### 4. 备份与迁移 💾
*   **备份**：点击顶部导航栏的 **“备份数据”**，浏览器会下载一个 `steam-monitor-backup.json` 文件。
*   **恢复**：点击 **“恢复数据”**，选择你之前下载的 JSON 文件，游戏列表瞬间恢复。

## ❓ 常见问题 (FAQ)

**Q: 运行 npm 时提示“无法加载文件...禁止运行脚本”？**
A: 这是 Windows PowerShell 的默认安全策略导致的。你可以任选一种方法解决：
*   **方法一（推荐）**：使用 **Command Prompt (CMD)** 代替 PowerShell 运行命令。
*   **方法二**：以管理员身份运行 PowerShell，输入 `Set-ExecutionPolicy RemoteSigned -Scope CurrentUser` 并回车确认。

**Q: 为什么添加游戏时提示“网络错误”？**
A: 本项目使用公共代理连接 Steam。如果网络不稳定，系统会自动重试。如果一直失败，请检查是否开启了 VPN，或者稍后再试。

**Q: 自动更新会卡死吗？**
A: 不会。我们内置了超时控制和错误吞没机制。即使某个游戏获取失败（如 Steam 接口抽风），系统也会跳过它继续执行，并在下一轮尝试修复，绝不会卡在某个进度。

**Q: 为什么看不到系统通知？**
A: 请确保浏览器允许了本网站的“通知权限”。通常在地址栏左侧有个锁形图标，点击后允许“通知”即可。
